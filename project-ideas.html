Project Idea

Summary

Email phishing attacks (baiting an unsuspecting computer user and hoping that they will bite) are some of the most frequent and dangerous online threats that target individuals and businesses these days.

Often being social engineering attacks, phishing emails can be cleverly crafted to appear completely benign technically to both user and antimalware scanner but tricking an email recipient into cliking on malicious site that looks legitimate or download a document that hides malware.

PhishNet is is phishing email analysis and open intelligence system designed to expedite email triage and analysis process as well as threat intelligence sharing.

Challanges and Motivation

Adversaries continuously invent new and creative ways to fool users into believing their actions involve a legitimate website or email by forging websites to appear identical to the real sites.
Combating ongoing and ever evolving phishing email threats typically requires a team of cyber security analysists, end user education, expensive email scanner platforms and malware sandboxes.

Our team #insert team name# believes we can help small businesses and individuals fight against fraud and phishing emails avoiding costly enterprise solutions, enriching community threat intelligence as well as being an open system providing and consuming external system APIs.

Image credit: Shutterstock

Three possible methods that can reduce incidents from occuring due to phishing attacks are:

  1. Detect and prevent phishing attacks from reaching recipient
  2. Detect and prevent after the user has been tricked into opening up a site url or attachment
  3. User education and tooling into indentifying potential threats
While all three above solutions are important to consider in combating phishing threats, User education and tooling is in our opinion the most important method shall the defences fail to prevent an attack programatically.

In a typical environment, an investment is made into some form of a solution that attempts at blocking a malicious email by means of:

  • Link scanning
  • Attachment scanning for viruses
  • Finding keywords identifyign spam and phishing

But is this enough to safeguard users? Not always. Security controls can fail, links can be cleverly obfuscated or be hosted on legitimate websites with good reputaiton but were compromised, malicious documents can be crafted to bypass antivirus detection.

Identifying the market

Now that the gap has been identified and demonstrated in above infographic, we present a solution aiming at addressing the specific problems of malicious emails getting through security controls which are to involve a user making an informed decision.

What are the project objectives:

  • Facilitate email analysis and context to help make better informed decisions when investigating emails
  • Integration with other security solutions to automate or at least augment phishing threat response
  • Valuable up to date phishing threat intelligence shared across users and automatic classification of email senders, links and attachments based on currently shared indicators.
  • Consequently as a result of using the platform, user becomes educated in understanding email technology and associated threats.

What the project is not intended to do:

  • Detect and Prevent phishing emails from reaching recipients.
    There are a number of mature commercial(Microsoft Defender, Forcepoint Email Gateways) and open source solutions (i.e. CLamAV, SpamAssassin)
    However, it is not possible to prevent a proportion of phishing emails to day with ever evolving techniques from adversaries.
    This means only the big market players with enough human resource and budget capacity can provide some form of protection in terms of evolving threat detection rules and intelligence.
  • Contain or remediate phishing email incidents.
    This type of functionality would largely depend on the users environment.
  • Make logical decisions on behalf of the user

Product description

This will be a web based application hosted on the cloud allowing users to sign up, upload emails they had received for analysis and option to have a privately analysed email or to share the analysis results with the community. Once user uploads email, it will be deconstructed; attachments and links extracted and scanned using open source scanners like VirusTotal.

All records will be kept in backend relational database (users login details, emails analysed, sender list, attachments, links).

Emails will be analysed and tagged as benign, anomalous, malicious accordingly by the system and user will be able to adjust the verdict tags,users will be also able to contribute to community by specifying if analysis had technical errors

Ongoing maintenance of this project would be largely crowd sourced in terms of intelligence, i.e. indicators of malicious senders, files, links, etc would be shared by analysists and users to the community.

Product features

  • Abilities given for users to register and create an account and to login to the application
  • Abilities given for users to upload email to auto-analyse an email for threats, sender authenticity
  • Safely preview the email via web interface and analyse links and attachments
  • Store email metadata, content and analysis results in database as normalized records
  • Extract and store email attachment for analysis
  • Scan attachments using REMnux Toolkit for Malware Analysts
  • Scan attachments and links using using external APIs (VirusTotal, Urlscan.io)
  • Abilities given for users to classify analysed emails as Benign, Spam, Phishing, Spear Phishing
  • Abilities given for users to search for analisys reportd
  • Use previously stored email data and classifications other users have made to relevant metadata and content in aws application to enhance accuracy in analysis of emails
  • Provide API endpoints for developers to query and update various metadata (i.e. malicious URLs, email domains, web domains).
  • Abilities given for users to mark a existing email analysis as false positive

PhishNet MVP functionality

  • Users forward to or upload emails to PhishNet system and initialize analysis
  • Any sensitive information like Personal Identifiable information is stripped from emails automatically
  • PhishNet automatically checks if the sender is authentic by verifying SPF, DMARC, DKIM records against sender email address and will automatically classify email as suspicious if not verified
  • PhishNet automatically checks sender address or domain, contained URLs and attachments against existing indicators already in the database and will automatically classify email as suspicious/malicious accordingly
  • If threats not found in existing records, PhishNet checks external APIs by hash of file, URL address and if found, will automatically classify email as suspicious/malicious accordingly
  • If threats not found in external intelligence records, scan URLs and attachments using toolkits and sandboxing and will automatically classify email as suspicious/malicious accordingly
  • User is able to store and share the analysis report with other PhishNet users
  • Users are able to query PhishNet analysis reports by means of searching and Dashboards
